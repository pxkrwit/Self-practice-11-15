ต้องใช้ try...catch เสมอเพื่อดักจับ error ทำให้โปรแกรมสามารถทำงานต่อได้เมื่อเกิด error
function ที่จะทำ CRUD ต้องเป็น async function เพราะต้องรอข้อมูลจาก backend
ต้องทำการ check status เพื่อเช็คว่าทำสำเร็จไหมตลอด

--- POST Method ---
- เป็น method ที่จะทำการส่ง request ไปหา backend เพื่อที่จะสร้างข้อมูลลงใน database
- โดย parameter ที่ต้องรับของ function ที่จะ POST Method มี 2 ตัว ( 1.รับ url ที่จะ fetch , 2.รับข้อมูลที่จะสร้างโดยจะรับเป็น JSON Object )
- สิ่งที่ต้องส่งไปด้วยเมื่อ fetch คือ method, headers, body
    - method : เพื่อบอกให้ backend รู้ว่าเราส่งมาด้วย method อะไร
    - headers : เพื่อกำหนดประเภทของข้อมูลที่จะส่งไป ในที่นี้ใช้เป็น JSON
    - body : ข้อมูลที่จะส่งไป ( อยู่ในรูปแบบของ JSON )
- เช็ค status ว่าเท่ากับ 201 ไหมถ้าไม่ใช่ให้ throw Error ( 201 คือ status ที่จะส่งกลับมาเมื่อสร้างสำเร็จ )
- สร้างตัวแปรเพื่อรอกับ response จาก backend
- return response ทีได้

--- PUT ---
- เป็น method ที่ใช้สำหรับการแก้ไขข้อมูลใน database 
- การส่ง request จะคล้ายกับ POST
- การที่จะ update ข้อมูลได้จำเป็นต้องมีข้อมูลชุดนั้นอยู่ใน database ก่อน 
- status ที่จะส่งต้องเป็น 200 ok เพื่อแสดงว่าเราทำการ update สำเร็จ

--- DELETE ---
- เป็น method สำหรับการลบข้อมูลใน database ออก
- การรับ parameter จะต้องรับ 2 ตัวเหมือนกับ POST และ PUT แต่อีกตัวที่รับจะเป็น id
- การที่จะลบได้จำเป็นต้องมี id ชุดนั้นอยู่ใน database ก่อนเหมือนกันถึงจะลบได้
- แต่ path ในการ fetch จะแตกต่างกันเพราะต้องทำการส่ง parameter ผ่าน url ไปด้วย
    - `${url}/${id}` ใช้แบบนี้
- และส่งไปด้วย method DELETE
- ใช้ res.ok เพื่อเช็ค status ว่า success ไหม ( res.ok = 200 ok)
- แล้วก็ทำงานคืนค่า id นั้นกลับ
